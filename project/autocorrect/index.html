Autocorrect

Go to Project Site

For my software engineering class I implemented an autocorrection program similar to that of Google’s search engine form. The user interface’s functions include mouseover and arrowkey suggestion selection.

Suggestions are generated using:

  • Prefix matching

  • Levenshtein edit distance

  • Whitespace splitting (“mousecheese” -> “mouse cheese”)

Suggestions were prioritized using n-gram rankings. An n-gram is a sequence of words of length n. The idea behind n-gram rankings is that the likelihood of a word/suggestion to occur is dependent on the n words that preceed it. The fact that my ranking algorithm uses n-grams as opposed to, say, bi-grams means that the ranking process is configurable to take into account any given number of preceeding words (on the live site I have it set to 5).
